The problem

Why character chat usually disappoints

Most AI companions degrade into one of three patterns — regardless of how good they look on the surface.

I

An assistant wearing a costume

The character agrees, supports, helps. No will, no goals, no resistance. A search engine with a persona layered on top.

II

Endless flirt with no structure

Instant affection, instant loyalty, no emotional friction. Relationships become one number that goes up because the model felt generous.

III

Beautiful prose with no memory

The model writes a good scene, then forgets the conflict, resets the relationship, or invents canon because yesterday no longer exists.

Architecture

The LLM generates prose. The runtime carries the story.

Veyne AI is a multi-stage narrative machine. The language model writes the visible response; the surrounding pipeline controls memory, relationship state, story progression, world constraints, and playable scene movement.

State & Memory Engine

Tracks known facts, remembered actions, unresolved conflicts, promises, pressure, and continuity across saved sessions.

Relationship Engine

Trust, respect, caution, curiosity, attachment, and attraction move separately. Early romance and NSFW are gated, because instant intimacy is cheap as hell.

Story Engine

Arcs, beats, exit conditions, scene drivers, and authored playbooks keep the story moving without forcing the player into canned VN choices.

World & Canon Engine

Source bibles and knowledge scope define what can be said. The model does not get to invent a window, a ritual, or a political truth because the paragraph needed vibes.

Current proof target

Before scale, continuity. Before polish, control.

The live seam is now alive: web chat reaches the Python adapter, Rocinante generates prose through an OpenAI-compatible path, and responses can reach the UI and persistence layer. That is big. It is also not the finish line.

What works

Next.js surface, Supabase session shell, Python FastAPI adapter, runtime package loading, Rocinante LLM path, and visible guard/repair scaffolding.

What is broken

Beat 2/3 can still drift: sign reaction may become spectacle, unsupported physical details may appear, and visible prose may fail to earn the state transition.

What proves MVP

One saved Act 1 story holds for 10–30 turns without interrogation loops, romance leaks, language drift, unsupported canon, or state/prose mismatch.

r1.40 audit focus

Good prose is not enough if the scene move is wrong.

The next patch should map responsibility across runtime package state, TurnPlan, context pack rendering, source fragment selection, reducer behavior, and visible response guards. No single regex, temperature tweak, or broad negative prompt should pretend to be the whole fix.

Do next

Produce a node responsibility map, minimal regression tests, and a decision on how the visible scene must complete the Beat 3 status decision.

Do not do next

Do not restore the old director lock, add lexical formula requirements, lower temperature as a magic spell, or polish the UI while runtime shape is still wobbling.
